### A.4 SQL block — Grouping backfill (6 upcoming, live excluded)

`Grouping` was copied from each tournament's source template (`TournamentTemplates.ConfigJson`). Live tournament 318866 was deliberately skipped — `ProcessGrouping` had already executed (no `BracketId` to assign retroactively), and `RatingMultiplier = 1.0` in all source templates means rating math is identical with or without `Grouping` in the live config.

```sql
BEGIN TRAN;

UPDATE t
SET ConfigJson = JSON_MODIFY(
        t.ConfigJson,
        '$.Grouping',
        JSON_QUERY(tt.ConfigJson, '$.Grouping')
    )
OUTPUT
    inserted.TournamentId,
    LEN(deleted.ConfigJson)  AS OldLen,
    LEN(inserted.ConfigJson) AS NewLen,
    JSON_VALUE(inserted.ConfigJson, '$.Grouping.MinSize')                 AS NewMinSize,
    JSON_VALUE(inserted.ConfigJson, '$.Grouping.Brackets[0].BracketName') AS NewBr1,
    JSON_VALUE(inserted.ConfigJson, '$.Grouping.Brackets[1].BracketName') AS NewBr2,
    JSON_VALUE(inserted.ConfigJson, '$.Grouping.Brackets[2].BracketName') AS NewBr3
FROM dbo.Tournaments t
JOIN dbo.TournamentTemplates tt WITH (NOLOCK) ON tt.TemplateId = t.TemplateId
WHERE t.TournamentId IN (318867, 318868, 318869, 318870, 318871, 318872)
  AND t.KindId = 3
  AND JSON_QUERY(tt.ConfigJson, '$.Grouping') IS NOT NULL;

-- Verify OUTPUT (6 rows; +~430 bytes per row; MinSize=20; Brackets Newbies/Midles/Tops with MinRating 0/101/1001)
COMMIT;
```

## B. Template 181 + Tournament 320415 — `Place 7` Rating fill

Content defect: `Place 7` had no `Rating` in template 181 ("Triple the Trout!"). Tournament 320415 (TemplateId 181, scheduled 2026-05-06 06:00 UTC) inherited the gap. Design provided `Rating = 13`. Both rows updated in a single transaction.

```sql
BEGIN TRAN;

UPDATE dbo.TournamentTemplates
SET ConfigJson = JSON_MODIFY(ConfigJson, '$.Places[6].Rating', 13)
OUTPUT 'Template' AS Side, inserted.TemplateId,
       LEN(deleted.ConfigJson)  AS OldLen,
       LEN(inserted.ConfigJson) AS NewLen,
       JSON_VALUE(inserted.ConfigJson, '$.Places[6].PlaceId') AS NewPlaceId,
       JSON_VALUE(inserted.ConfigJson, '$.Places[6].Rating')  AS NewRating
WHERE TemplateId = 181
  AND JSON_VALUE(ConfigJson, '$.Places[6].PlaceId') = '7'
  AND JSON_VALUE(ConfigJson, '$.Places[6].Rating') IS NULL;

UPDATE dbo.Tournaments
SET ConfigJson = JSON_MODIFY(ConfigJson, '$.Places[6].Rating', 13)
OUTPUT 'Tournament' AS Side, inserted.TournamentId,
       LEN(deleted.ConfigJson)  AS OldLen,
       LEN(inserted.ConfigJson) AS NewLen,
       JSON_VALUE(inserted.ConfigJson, '$.Places[6].PlaceId') AS NewPlaceId,
       JSON_VALUE(inserted.ConfigJson, '$.Places[6].Rating')  AS NewRating
WHERE TournamentId = 320415
  AND KindId = 3
  AND JSON_VALUE(ConfigJson, '$.Places[6].PlaceId') = '7'
  AND JSON_VALUE(ConfigJson, '$.Places[6].Rating') IS NULL;

-- Verify OUTPUT (2 rows: Template + Tournament, NewPlaceId=7, NewRating=13)
COMMIT;
```

## C. FinishPoint normalisation (15 templates)

Content defect: 15 active templates carry an RFC-invalid `FinishPoint` block — JavaScript object literal with unquoted property names: `{ X: 307, Y: 0, Z: 596 }`. Pattern was identical across all 15 (whitespace `{ X:` / `, Y:` / `, Z:`). Fixed via idempotent `REPLACE`.

### C.2 SQL block

```sql
UPDATE dbo.TournamentTemplates
SET ConfigJson =
    REPLACE(REPLACE(REPLACE(
        ConfigJson,
        '{ X:', '{ "X":'),
        ', Y:', ', "Y":'),
        ', Z:', ', "Z":')
WHERE ConfigJson LIKE N'%FinishPoint%'
  AND ISJSON(ConfigJson) = 0;
```

Result: 15 rows affected, 295 ms. Verified post-patch: `HasFinishPoint=15`, `StillInvalid=0`, `NowValid=15`, no residual `{ X:` / `, Y:` / `, Z:` markers anywhere in `TournamentTemplates`.
